Chatovací aplikace v Pythonu

Jednoduchý začátečnický projekt pro procvičení komunikace klient-server

O projektu

Jednoduchá chatovací aplikace postavená v Pythonu, která umožňuje více uživatelům komunikovat v reálném čase prostřednictvím příkazového řádku. Aplikace využívá TCP sockety pro spolehlivou komunikaci a vlákna (threading) pro souběžné zpracování více klientů. Projekt beru jako ukázku základů práce se sítí, ne jako hotový produkt.

Architektura aplikace

Aplikace funguje na principu klient-server:

Klient 1
↔
Server
↔
Klient 2
Klient 3
↔
Server
↔
Klient N
  • Server – přijímá připojení klientů, přeposílá zprávy všem ostatním
  • Klient – připojí se k serveru, odesílá a přijímá zprávy

Struktura souborů

Soubor Popis
server.py Chat server – přijímá připojení a přeposílá zprávy
client.py Chat klient – uživatelské rozhraní v terminálu

Návod k použití

Požadavky

  • Python 3.6 nebo novější
  • Žádné externí knihovny – pouze standardní knihovna Pythonu

Krok 1: Spuštění serveru

Otevřete terminál a přejděte do složky chat/:

$ cd chat
$ python server.py

[Server] Chat server běží na 127.0.0.1:65432
[Server] Čekám na připojení klientů...

Krok 2: Spuštění klienta

Otevřete nový terminál (server musí zůstat spuštěný):

$ python client.py

========================================
   Chatovací aplikace v Pythonu
   (Socket Programming)
========================================

Zadejte svou přezdívku: Marek
[Klient] Připojeno k serveru 127.0.0.1:65432

Krok 3: Chatování

Napište zprávu a stiskněte Enter. Zpráva se zobrazí všem ostatním klientům:

# Klient 1 (Marek)
Ahoj, jak se máš?

# Klient 2 (Jana) vidí:
[Marek] Ahoj, jak se máš?

Krok 4: Odpojení

  • Napište /quit pro ukončení klienta
  • Na serveru stiskněte Ctrl+C pro zastavení serveru

Funkce aplikace

  • Více klientů najednou – server obslouží několik připojených uživatelů
  • Přezdívky – každý uživatel si zvolí jméno při připojení
  • Zprávy v reálném čase – zprávy se doručují okamžitě všem klientům
  • Oznámení – server informuje o připojení a odpojení uživatelů
  • Základní práce s vlákny – každý klient běží ve vlastním vlákně
  • Jednoduché ukončení – klient i server lze korektně zastavit

Klíčové koncepty

  • TCP Sockety – spolehlivé, uspořádané doručování dat
  • Threading – každý klient běží ve vlastním vlákně
  • Broadcasting – server přeposílá zprávu všem kromě odesílatele
  • UTF-8 kódování – podpora českých a slovenských znaků
